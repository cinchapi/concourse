# concourse-unit-test-core
An extension of the [JUnit](http://junit.org/) framework that provides scaffolding for all Concourse unit tests.

## Usage
All unit tests should extend `ConcourseBaseTest` to take advantage of logic that will log execution state whenever a test fails.

### Registering Execution State
The `Variables` class can be used to track the execution state for each test.
```java
long number = Variables.register("number", TestData.getLong());
```

The state is cleaned up at the beginning of each test. If a test fails, all the Variables are automatically dumped to the console log for debugging:
